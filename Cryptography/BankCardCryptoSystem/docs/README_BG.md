## Преглед
Системата за криптиране на банкови карти осигурява сигурно криптиране и декриптиране на номера на кредитни карти чрез заместващ шифър. Системата поддържа множество клиенти, свързани към многонишков сървър или RMI сървър.

## Възможности
- **Двойна имплементация на сървъра**: Избор между многонишков сокет базиран или RMI базиран сървър
- **Сигурно удостоверяване**: Контрол на достъпа въз основа на роли с потребителски идентификационни данни
- **Валидиране на карти**: Имплементира алгоритъма на Luhn за валидиране на номера на карти
- **Криптиране**: Заместващ шифър с конфигурируема стойност на отместване
- **Одит логване**: Проследява всички операции за криптиране/декриптиране
- **Управление на потребители**: Административен интерфейс за управление на потребители и права

## Архитектура на системата

### Компоненти
1. **Общ модул**
   - Споделени модели от данни (Потребител, Данни за карта)
   - Инструменти за криптиране/декриптиране
   - Логика за валидиране на карти

2. **Многонишков сървър**
   - Обработва едновременни клиентски връзки
   - Група нишки за ефективно използване на ресурсите
   - Административен интерфейс с JavaFX

3. **RMI сървър**
   - Интерфейс за отдалечени извиквания на методи
   - Централизирана имплементация на услугата
   - Административен интерфейс с JavaFX

4. **Клиентско приложение**
   - Потребителски интерфейс, базиран на JavaFX
   - Поддръжка и на двете имплементации на сървъра
   - Валидиране на входните данни и обработка на грешки

## Инсталация

### Изисквания
- Java 11 или по-нова версия
- Maven 3.6+

### Инструкции за изграждане
```bash
# Клониране на хранилището
git clone <repository-url>

# Изграждане на проекта
mvn clean install
```

## Употреба

### Стартиране на сървъра
```bash
# За многонишков сървър
java -jar multithreaded-server/target/multithreaded-server-1.0.jar

# За RMI сървър
java -jar rmi-server/target/rmi-server-1.0.jar
```

### Стартиране на клиента
```bash
java -jar client/target/client-1.0.jar
```

## Сигурност
- Всички пароли се съхраняват хеширани
- Валидиране на входните данни от страна на клиента и сървъра
- Контрол на достъпа въз основа на роли
- Сигурно управление на сесиите

## Често срещани проблеми
### Основни проблеми
1. **Връзката е отказана**
   - Проверете дали сървърът работи
   - Проверете настройките на защитната стена
   - Уверете се, че номерата на портовете са верни

2. **Грешки при удостоверяване**
   - Проверете потребителското име и паролата
   - Проверете правата на потребителя
   - Уверете се, че акаунтът не е заключен

3. **Грешки при валидиране на карта**
   - Проверете формата на номера на картата
   - Проверете съответствието с алгоритъма на Luhn
   - Уверете се, че префиксът на типа карта е правилен (3,4,5,6)
